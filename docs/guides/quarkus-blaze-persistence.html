Quarkus Blaze Persistence

Propiedad de configuración fijada en tiempo de compilación - Todas las demás propiedades de configuración son modificables en tiempo de ejecución

Propiedad de configuración

Tipo

Por defecto

quarkus.blaze-persistence.template-eager-loading

Una bandera booleana para hacer posible la preparación de todos los cachés de las plantillas de vista en el arranque. Por defecto, la carga anticipada de las plantillas de vista está desactivada para tener un mejor rendimiento en el arranque. Los valores válidos para esta propiedad son true o false.

booleano

quarkus.blaze-persistence.default-batch-size

Un valor entero que define el tamaño del lote por defecto para los atributos de la vista de entidad. Por defecto el valor es 1 y puede ser modificado a través de com.blazebit.persistence.view.BatchFetch#size() o estableciendo esta propiedad a través de com.blazebit.persistence.view.EntityViewSetting#setProperty.

int

quarkus.blaze-persistence.expect-batch-mode

Un modo que especifica si se espera el valor de correlación, la raíz de la vista o el loteo de la vista incrustada. Por defecto el valor es values y puede ser anulado estableciendo esta propiedad mediante com.blazebit.persistence.view.EntityViewSetting#setProperty. Los valores válidos son - values - view_roots - embedding_views

cadena

quarkus.blaze-persistence.updater.eager-loading

Una bandera booleana para hacer posible la preparación de la caché del actualizador de la vista de entidad en el arranque. Por defecto, la carga anticipada de las actualizaciones de la vista de entidad está desactivada para tener un mejor rendimiento en el arranque. Los valores válidos para esta propiedad son true o false.

booleano

quarkus.blaze-persistence.updater.disallow-owned-updatable-subview

Una bandera booleana para hacer posible la desactivación de la validación estricta que no permite el uso de un tipo de vista de entidad actualizable para las relaciones de propiedad. Por defecto el uso está deshabilitado, es decir, el valor por defecto es true, pero como puede haber modelos extraños, es posible permitirlo. Los valores válidos para esta propiedad son true o false.

booleano

quarkus.blaze-persistence.updater.strict-cascading-check

Un indicador booleano que permite desactivar la comprobación de cascada estricta que impide establecer vistas de entidad actualizables o creables en atributos no cascables antes de asociarlas a un atributo cascable. Cuando se desactiva, es posible, como en JPA, que los cambios realizados en una vista de entidad actualizable no se vuelquen cuando no está asociada a un atributo que actualiza en cascada. Por defecto el uso está habilitado, es decir, el valor por defecto es true. Los valores válidos para esta propiedad son true o false.

booleano

quarkus.blaze-persistence.updater.error-on-invalid-plural-setter

Un indicador booleano que permite cambiar de advertencias a errores de validación en el momento del arranque cuando se encuentran definidores de atributos plurales no válidos mientras la comprobación estricta en cascada está activada. Cuando es true, se lanza un error de validación en tiempo de arranque cuando se encuentra un definidor inválido, de lo contrario sólo una advertencia. Esta configuración no tiene efecto cuando la comprobación estricta en cascada está desactivada. Por defecto el uso está deshabilitado, es decir, el valor por defecto es false. Los valores válidos para esta propiedad son true o false.

booleano

quarkus.blaze-persistence.create-empty-flat-views

Un indicador booleano que permite especificar si las vistas planas vacías deben crearse por defecto si no se especifica mediante EmptyFlatViewCreation. Por defecto, la creación de vistas planas vacías está habilitada, es decir, el valor por defecto es true. Los valores válidos para esta propiedad son true o false.

booleano

quarkus.blaze-persistence.expression-cache-class

La expresión completa del nombre de la clase de implementación de la caché.

cadena

quarkus.blaze-persistence.inline-ctes

Si se establece como true, las consultas CTE se alinean por defecto. Los valores válidos para esta propiedad son true, false o auto. El valor por defecto es true, que siempre alineará las CTEs no recursivas. La configuración auto sólo hará uso del inlining si el proveedor JPA y el dialecto del DBMS lo soportan/requieren. La propiedad puede ser cambiada para un constructor de criterios antes de construir una consulta.

booleano
